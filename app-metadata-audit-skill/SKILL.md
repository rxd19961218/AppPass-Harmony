# App Metadata Audit Skill

当开发者通过AGC提交应用时，使用此 Skill 对填写的信息进行“第一轮预审”。

## 1. 核心审查逻辑 (Audit Logic)

该 Skill 运行时将扫描本地元数据文件（如 `metadata/*.json` 或 `.strings`），并对照以下维度进行风险分级：
- 🔴 **BLOCKER (阻碍)**: 绝对会被打回的错误（如包含竞品名称、链接失效）。
- 🟡 **WARNING (警告)**: 可能导致降权或审核延时的建议（如字符数接近上限、缺少关键词）。
- 🟢 **INFO (提示)**: 优化建议（如描述中未包含核心功能词）。

## 2. 审查清单定义 (Checklist)

### 通用准则 (General)
- **信息的定义**: “信息”包含名称、图标、介绍（包括一句话简介、新版本特性）、应用截图和视频、分类及标签、语言、内容分级、开发者信息、隐私信息（如隐私政策、权限说明等）；“信息”需满足以下要求。
- **名称不得为泛词**: 名称不得为广义归纳、普遍且不具辨识性的词汇或热门搜索词，避免干扰搜索结果及误导用户，包括但不限于使用商标术语、热门应用名称或别称、流行词、行业名词、职业名词、类别词、功能性描述的词汇，以及在名称中堆砌2个或以上关键词，如手机定位、手电筒、日历、视频剪辑去水印修图等。
- **信息不得含热词**: 信息中不得含有热搜词，名称不得堆砌多个热词，不得蹭热度误导用户。
- **名称字符限制**: 名称长度不得超过15个汉字字符或30个其他语言字符。
- **信息不得侵权**: 名称不得和知名应用名称、IP相同或相似，信息不得包含知名应用名称、IP内容元素。
- **名称不得夸大宣传**: 名称不得包含定价信息、价格、诱导赚钱等具有营销属性的商业化词汇（如免费、促销、清仓、XX元、￥XX、9块9、躺赚等），避免夸大宣传、误导用户。
- **名称不得含有占位符**: 名称不得含有占位符文本、乱码、表情符号、特殊符号 (如“*” “&”“-”“( )”) 。
- **图标不得为系统图标**: 图标不得为系统图标，避免被用户误认为是系统应用。
- **图标不得含误导点击元素**: 图标不得包含误导用户点击的元素，如在没有新消息的情况下显示代表有新消息的数字角标。
- **信息使用语言需与选择一致**: 需提供支持本地化的语言版本，信息及内容使用的语言需与选择的语言一致。
- **名称含义需一致**: 各个语言环境下名称含义需一致。
- **信息需与内容相符**: 信息需与内容相符，不得包含误导性、不相关或不恰当的商业化用语、热门搜索词、流行词语，且不得宣传实际并不提供的内容或服务、以及做出无法证实的产品声明。信息不得误导或暗示应用的排名和表现，如使用官方、权威、推荐、精品等词汇。
- **信息不得含误导内容**: 信息不得在未经授权的情况下使用或者变相使用代表政府实体或其他知名实体的标识及内容，如：非国家机构使用国旗、国徽、党旗、党徽等标识作为图标；不得未经批准使用“电视台”、“广播电台”、“电台”、“TV”等广播电视专有名称开展业务。
- **信息不含其他终端品牌**: 信息不得带有其他终端品牌或移动应用平台的名称、图标或图像。
- **信息不得含低俗色情等违法内容**: 信息不得存在低俗、性暗示、色情、暴力、恐怖、血腥、赌博、非法金融活动、非法虚拟专用网络VPN跨境经营活动及其他适用法律法规所禁止的内容。
- **信息不得含儿童不宜内容**: 信息不应含有可能对儿童造成负面影响的元素；二级分类为“儿童”的年龄分级不得大于12+，二级分类为“儿童”的才可以在信息中使用表示主要受众为儿童的内容，如：“适用幼儿/儿童”、“适合幼儿/儿童”，”专为幼儿/儿童设计” 等
- **信息不得展示个人信息**: 信息不得展示真实的个人信息，如个人真实账号、身份信息、电话号码等。
- **信息不得重复**: 一句话简介、介绍不得存在重复。
- **需提供测试账号**: 如需要账号才能使用，应提交有效可用的账号和登录信息。
- **隐私政策网址需能正常打开**: 隐私政策网址需能正常打开。
- **隐私政策需与个人信息保护相关**: 需提供开发者自身的隐私政策，隐私政策需与个人信息保护相关，需包含收集个人信息的目的、方式、范围；隐私政策需与提交的隐私政策网址内容保持一致。
- **隐私政策需易于访问和阅读**: 隐私政策网址和隐私政策需易于访问和阅读；需提供中文版的隐私政策；隐私政策等收集使用规则不得难以阅读，如文字过小过密、颜色过淡、模糊不清、冗长文本，展示不全。


### 华为应用额外审核规范 (HarmonyOS)
- **截图和视频需清晰完整**: 应用截图和视频上传时需选择正确的方向，应用截图需至少上传3张不同内容的图片，不得出现重复、模糊、拉伸、压缩、显示不完整、无法体现实际内容等情况。
- **应用需提供测试录屏**: 若应用内的功能需要连接设备或需要特殊的运行环境等场景才能使用，需提供演示视频。
### 华为元服务额外审核规范 (HarmonyOS)
无额外规范，参考 **通用准则 (General)**


## 3. 工作流指令 (Workflow)

### 步骤 1：本地数据准备
AI 将查找由`agcli` 拉取到本地的目录：
- `./metadata/` (默认)
- `./agc-locales/`

### 步骤 2：启动审查
开发者可以直接下令：
> **“按照 app-metadata-audit-skill 规范，审查我当前的华为应用元数据，并输出一份审计报告。”**

### 步骤 3：报告格式
AI 将按以下格式返回：

| 字段   | 状态 | 问题说明            | 建议修改方案            |
|:-----| :--- |:----------------|:------------------|
| 应用名称 | 🟢 | 正常              | -                 |
| 简短描述 | 🔴 | 包含敏感词 "Android" | 建议改为 "适用于移动设备的版本" |
| 简介问题 | 🟡 | 一句话简介、介绍重复      | 建议优化修改一句话介绍       |

## 4. 自动修复 (Autofix)
该 Skill 必须具备修改本地文件的能力：
- `fix-length`: 自动处理超长字符至限制范围，对内容做修改和和摘要。
- `remove-competitors`: 自动查找并替换竞品词汇。

## 5. 扩展建议 (Guardrails)
- **环境隔离**: 审查不应直接修改线上数据，仅修改本地待上传文件。
- **规范更新**: 定期同步华为最新的[审核指南FAQ](https://developer.huawei.com/consumer/cn/doc/app/50106)。
